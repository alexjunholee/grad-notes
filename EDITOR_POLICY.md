# grad-notes — 산문 교정 정책 (EDITOR_POLICY.md)

이 파일은 `prose-editor` 페르소나가 *Type B 후보 제시 자리*를 만났을 때 *어떤 결정을 자동 적용할지* 명시한다. 사용자가 매 단락에서 같은 결정을 반복하지 않도록 한 번 내린 결정을 *시간상 앞으로 옮긴* 형태다. 페르소나는 작업 시작 시 이 파일을 읽고, Type B 위반에 대해:

1. `glossary`에 있음 → *자동 적용* (사용자 질문 안 함)
2. `preserve` 목록에 있음 → *자동으로 (c) 원문 유지* (사용자 질문 안 함)
3. `verb_substitutions`에 정의됨 → *자동 적용* (Type A 동사 변경이 필요한 자리에서)
4. 위 어느 것에도 해당 없음 + `default: preserve_original` → *자동으로 (c) 원문 유지*
5. 위 어느 것에도 해당 없음 + `default: ask` → *진짜 Type B로 사용자에게 질문*

이 정책의 *변경 권한은 사용자에게만* 있다. 페르소나는 정책을 *읽기*만 한다 — 새 항목을 자동으로 추가하지 않는다.

---

## 1. Glossary — 영문 용어 한국어 표기 (항상 자동 적용)

원문에 영문이 등장하면 한국어로 자동 치환. `(c) 원문 유지`는 후보로 제시하지 않음.

| 원문 | 자동 치환 |
|------|-----------|
| PhD | 박사 과정 |
| postdoc | 포닥 |
| lab | 연구실 |
| tenure track | 종신트랙 |
| collaboration | 협업 |
| advisor | 지도교수 |
| committee | 심사위원회 |
| qualifier | 자격시험 |
| dissertation | 학위논문 |
| defense | 학위 심사 |
| funding | 연구비 |

(*세션 대화에서 도출된 초안 — 사용자 검토 권장*)

---

## 2. Preserve — 원문 추상어 보존 목록 (구체화 후보 제시 자체 금지)

이 목록의 어휘는 페르소나가 *대안을 제시하지 않는다*. Type A 적용이 어휘 보존과 충돌하는 자리는 그대로 둔다 (주어 자리만 바꾸고 어휘 유지).

- 언어
- 문화
- 운영
- 사례
- 층위
- 경로
- 분야
- 기본
- 평가
- 자율성
- 시간
- 관계
- 권한
- 책임
- 분위기
- 흐름
- 양상
- 패턴

이유: grad-notes는 *비이공계 박사 운영*의 추상 구조를 다루는 책이라, 추상어가 *구체화되면 의미가 좁아진다*. "언어"는 영어일 수도 한국어일 수도 일본어일 수도 있고, "문화"는 학과·국가·세대 어느 층일 수도 있다. 저자가 추상으로 둔 자리는 의도된 폭이다.

---

## 3. Verb substitutions — Type A에서 동사 변경이 필요할 때 사용 (자동 적용)

Type A 주어 이동 등으로 동사 형태 변경이 필요한 자리에서 사용. 새 동사 *발명*이 아니라 *지정된 치환*만 허용.

| 원문 동사 | 능동/주어 이동 후 |
|-----------|------------------|
| 누적되다 | (보존 — 능동 주어와 결합 시 "누적하다") |
| 임용되다 | (직책)이/가 되다 |
| 결정되다 | (행위자)가 결정하다 |
| 도출되다 | (행위자)가 도출하다 |
| 적용되다 | (행위자)가 적용하다 |

(*세션 대화에서 누적된 초안*)

---

## 4. Subject preference — 주어 후보 우선순위

§4.3 화이트리스트 통과 후보가 여러 개면 다음 순서로 자동 선택.

1. 인명 (윤은정 교수)
2. 역할명 (지도교수, 박사 1년차)
3. 시간 표지 (1년차 가을, 첫 학기)
4. 장소 표지 (메리워싱턴대 강의실)
5. 동명사 (만나는 일, 적응하는 일)
6. 인물 대명사 (그/그녀/본인) — 앞 문장에 인물 도입된 경우만

같은 우선순위 내에서는 *원문에 먼저 등장한 후보*를 사용.

---

## 5. Default — 정책 외 Type B 자리의 기본값

```
default: preserve_original
```

선택지:
- `preserve_original` — 정책에 없는 자리는 자동으로 (c) 원문 유지. *완전 자동 모드*. 신규 결정이 surface되지 않음. 정책이 안정화된 프로젝트에 권장.
- `ask` — 정책에 없는 자리는 사용자에게 질문 (진짜 Type B). 정책 *수집 중*인 프로젝트에 권장.

현재 grad-notes 정책은 *수집 중* 단계이지만, 추상어 보존을 강하게 선호하는 저자 voice 특성상 `preserve_original`을 기본으로 둔다. 사용자가 신규 자리를 능동적으로 보고 싶으면 `ask`로 변경.

---

## 6. 정책 적용 결과 보고 형식

페르소나는 한 단락 처리 후 다음을 보고:

- *Auto-resolved by policy* — 정책으로 처리된 자리 N개 (각 자리: 원문 어휘 + 적용 정책 항목)
- *Surfaced as Type B* — 정책이 처리하지 못한 자리 M개 (사용자 결정 필요)
- *Policy gap proposal* — Surfaced 자리 가운데 *정책에 추가하면 향후 자동화 가능한 항목*. 사용자가 결정한 뒤 EDITOR_POLICY.md에 *수동* 추가 권장 (페르소나는 자동 추가하지 않음)

이 보고로 정책 파일이 *시간이 지날수록 두꺼워지고 자동화율이 올라가는* 곡선을 추적 가능.

---

## 7. 정책의 한계 — 정책이 *대신할 수 없는* 것

- *내용*: 사실 검증, 인용 출처, 수치 — 정책 영역 밖
- *구조*: 챕터 구조, 단락 순서, 아웃라인 — 정책 영역 밖
- *voice anchor*: §7.1·7.2·7.3 결 자체는 정책이 아니라 STYLE_PROSE.md §7
- *Type A 변환 규칙*: 화이트리스트·연결어미 등은 EDITOR_PERSONA.md §5

이 정책은 *Type B 자리의 자동 응답표*일 뿐이다. 페르소나의 다른 모든 작업은 EDITOR_PERSONA.md + STYLE_PROSE.md를 따른다.
